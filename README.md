# ThebeRTOS-AVR

## Professional Preemptive Mini-RTOS for ATmega328P

ThebeRTOS-avr is a lightweight, bare-metal preemptive real-time operating system (RTOS) designed specifically for the ATmega328P microcontroller. This kernel demonstrates professional-grade embedded architecture, incorporating advanced synchronization primitives, real-time diagnostics, and deterministic scheduling.

## Core Features

### Hybrid Priority-Based Scheduler
*   **Preemptive Model**: Ensures that the highest-priority "READY" task is always executing.
*   **Round-Robin Logic**: Employs round-robin scheduling for tasks sharing the same priority level to ensure equitable CPU distribution.
*   **Minimal Latency**: Core context switching is implemented in optimized Assembly.

### Synchronization & IPC
*   **Priority Inheritance Mutexes**: Prevents priority inversion by temporarily boosting the priority of a mutex owner when a higher-priority task is pending.
*   **Blocking Message Queues**: Thread-safe Inter-Task Communication (ITC) using circular buffers. Supports "copy by value" semantics to ensure data integrity across task boundaries.
*   **Interrupt-Safe Semaphores**: Binary semaphores designed for signaling tasks from hardware interrupts (ISRs).

### System Reliability & Diagnostics
*   **Stack Watermarking**: Sophisticated "Stack Painting" (0xAA pattern) and runtime analysis to monitor per-task stack headroom and prevent overflows.
*   **Heap Monitoring**: Real-time tracking of available SRAM and heap boundaries.
*   **Software Timers**: Managed timer engine for executing non-blocking callbacks in the system tick context.
*   **System Health Monitor**: A dedicated background supervisor task that provides periodic UART reports on kernel state and memory health.

## Technical Architecture

The kernel operates on a 1 ms system tick generated by Timer1. The context switching mechanism manually preserves all 32 general-purpose registers and the Status Register (SREG) to ensure absolute state integrity between task transitions.

### Memory Footprint
*   Highly optimized for 8-bit AVR architecture.
*   Minimal SRAM overhead per Task Control Block (TCB).
*   User-defined static buffers for task stacks and message queues.

## Directory Structure

*   **/src/kernel**: Core kernel, scheduler, synchronization, and queue logic.
*   **/src/hal**: Hardware Abstraction Layer including Timer and UART drivers.
*   **/include**: System-wide header definitions and kernel APIs.
*   **/src/main.c**: Multi-tasking demonstration code.

## Getting Started

### Prerequisites
*   AVR-GCC Toolchain
*   GNU Make
*   avrdude (for hardware deployment)

### Build and Flash
1.  Connect your ATmega328P / Arduino Uno.
2.  Open the Makefile and verify the `DEVICE` port (e.g., COM3 or /dev/ttyUSB0).
3.  Execute the following commands:

```bash
make
make flash
```
